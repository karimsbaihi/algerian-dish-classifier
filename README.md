# Algerian Dish Classifier 🇩🇿

A computer vision model that recognises 15 traditional Algerian dishes from a photo.

**Live demo:** https://huggingface.co/spaces/karimsuba/algeriadish
**Validation accuracy:** **88.3%** (ResNet-50, transfer learning)

Built end-to-end as a shipped ML project: scraped and hand-cleaned a custom dataset, trained a ResNet-50 with two-phase transfer learning on a free Colab GPU, served it through a Gradio app, deployed publicly on Hugging Face Spaces — no paid APIs, no proprietary data.

---

## The 15 dishes

`couscous`, `chakhchoukha`, `rechta`, `mhadjeb`, `bourek`, `chorba_frik`, `harira`, `dolma`, `tajine_zitoune`, `karantika`, `mhalbi`, `makroud`, `baghrir`, `zlabia`, `kalb_el_louz`

Covers the full Algerian repertoire — soups (`harira`, `chorba_frik`), savoury mains (`couscous`, `tajine_zitoune`, `dolma`, `mhadjeb`), pastries and sweets (`bourek`, `makroud`, `zlabia`, `kalb_el_louz`), and street food (`karantika`, `baghrir`).

---

## Approach

1. **Data collection.** Scraped candidate images per class via DuckDuckGo's image search (`scrape_images.py`). Two scraper dead-ends along the way: `icrawler`'s Google parser is broken with current Google HTML, and Bing's async endpoint returns *trending filler* content for niche queries — both produced wildly off-topic results for obscure regional dishes. DuckDuckGo backed by Bing's real search API worked.
2. **Cleaning.** Manual cull of each class — wrong dish, Moroccan/Tunisian lookalikes, text and logos, duplicates, cartoons. Landed at ~100 images per class, balanced.
3. **Training.** ResNet-50 transfer learning on a free Colab GPU (`train_resnet50.ipynb`):
   - **Phase 1:** freeze the backbone, train only the new 15-class head (Adam, lr 1e-3, 8 epochs).
   - **Phase 2:** unfreeze and fine-tune the whole network (Adam, lr 1e-4, cosine schedule, 12 epochs).
   - Heavy regularisation against overfit on a small dataset: RandAugment, RandomErasing, label smoothing, weight decay.
4. **Iteration.** Baseline run hit a ceiling of 84.3%. Diagnosed it as a *data* ceiling (`chakhchoukha`, `karantika`, `dolma` had the worst recall), not a model ceiling. Targeted re-scrape and re-clean of those three classes (`rescrape_weak.py`) lifted the model to 88.3% — no architecture or hyperparameter changes.
5. **Serving.** Gradio app (`app.py`) loads the checkpoint, accepts an uploaded photo, returns the top-3 predictions with probabilities. Themed in Algerian flag colours.
6. **Deploying.** Pushed to a Hugging Face Space (`deploy_hf.py`) on free CPU hardware.

## Results

**Validation accuracy: 88.3%** on a stratified 80/20 split of the cleaned dataset.

| class            | precision | recall | f1   |
| ---------------- | --------- | ------ | ---- |
| baghrir          | 0.96      | 0.98   | 0.97 |
| bourek           | 0.95      | 1.00   | 0.97 |
| chakhchoukha     | 0.94      | 0.81   | 0.87 |
| chorba_frik      | 0.89      | 0.80   | 0.84 |
| couscous         | 0.85      | 0.85   | 0.85 |
| dolma            | 0.93      | 0.98   | 0.95 |
| harira           | 0.80      | 0.80   | 0.80 |
| kalb_el_louz     | 0.87      | 1.00   | 0.93 |
| karantika        | 0.90      | 0.85   | 0.87 |
| makroud          | 1.00      | 0.80   | 0.89 |
| mhadjeb          | 0.91      | 0.95   | 0.93 |
| mhalbi           | 0.73      | 0.80   | 0.76 |
| rechta           | 0.83      | 0.75   | 0.79 |
| tajine_zitoune   | 0.67      | 0.90   | 0.77 |
| zlabia           | 0.94      | 0.85   | 0.89 |
| **macro avg**    | **0.88**  | **0.87** | **0.87** |

The dishes the model still mixes up are the genuinely confusable ones — `harira` vs `chorba_frik` are both Algerian soups, `tajine_zitoune` vs `rechta` share the same chicken-and-sauce visual base. Honest limitation worth naming rather than papering over.

A confusion matrix is generated by the training notebook and saved to `confusion_matrix.png`.

## Repository layout

```
.
├── app.py                       # Gradio app served on Hugging Face Spaces
├── train_resnet50.ipynb         # Colab notebook — full training pipeline
├── scrape_images.py             # DuckDuckGo image scraper
├── rescrape_weak.py             # Targeted re-scrape for the weak classes
├── deploy_hf.py                 # Push app + model to the Hugging Face Space
├── requirements.txt             # Runtime deps (pinned to the build I tested)
├── data/                        # Local dataset (gitignored — never pushed)
└── models/                      # Trained checkpoint (gitignored — lives in the Space)
```

The dataset and the model weights are deliberately kept out of git. The model lives in the Hugging Face Space; the dataset stays local to respect bandwidth and the source images' provenance.

---

## Run it locally

```bash
pip install -r requirements.txt
# put resnet50_algerian_dishes.pth into ./models/
python app.py
```

The Gradio UI opens at http://127.0.0.1:7860.

To retrain from scratch: zip the `data/` folder, upload to your Google Drive, open `train_resnet50.ipynb` in Colab with a GPU runtime, and run top to bottom. The notebook handles the Drive mount, the split, the two-phase training and the per-class evaluation.
